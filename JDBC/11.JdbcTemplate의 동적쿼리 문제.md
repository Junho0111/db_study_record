# 1. JdbcTemplate - 동적 쿼리 문제

## 개요
* `JdbcTemplate`의 가장 큰 **단점**: 동적 쿼리를 해결하기 어려움
* 실무에서는 사용자의 검색 조건에 따라 SQL이 동적으로 변경되어야 함
* 이를 해결하기 위해 복잡한 문자열 연결 로직이 필요함

<br>

## JdbcTemplate 동적 쿼리 문제

### 동적 쿼리 예시

상황 예시 : 사용자의 검색 조건에 따라 DB로 날리는 쿼리문이 달라짐

* **검색 조건이 없는 경우**
```sql
select id, item_name, price, quantity
from item
```

<br>

* **상품명(`itemName`)으로 검색하는 경우**
```sql
select id, item_name, price, quantity
from item
where item_name like concat('%',?,'%')
```

<br>

* **최대 가격(`maxPrice`)으로 검색하는 경우**
```sql
select id, item_name, price, quantity 
from item
where price <= ?
```

<br>

* **상품명과 최대 가격 **둘 다** 검색하는 경우**
```sql
select id, item_name, price, quantity 
from item
where item_name like concat('%',?,'%') and price <= ?
```

<br>

### 동적 쿼리의 어려움

| 문제점 | 설명 |
| --- | --- |
| **조건부 WHERE 절** | `WHERE`를 언제 붙일지, `AND`를 언제 붙일지 판단해야 함 |
| **파라미터 순서** | 조건에 따라 동적으로 변경되는 파라미터 순서를 맞춰야 함 |
| **복잡한 로직** | 실무에서는 5개 이상의 검색 조건이 흔히 존재 |
| **유지보수의 어려움** | 조건이 추가될 때마다 코드 수정 필요 |

<br>

### JdbcTemplate에서의 동적 쿼리 예시

```java
@Override
public List<Item> findAll(ItemSearchCond cond) {
    String itemName = cond.getItemName(); 
    Integer maxPrice = cond.getMaxPrice();
    String sql = "select id, item_name, price, quantity from item";
    
    if (StringUtils.hasText(itemName) || maxPrice != null) {
        sql += " where";
    }
    
    boolean andFlag = false;
    List<Object> param = new ArrayList<>();
    
    if (StringUtils.hasText(itemName)) {
        sql += " item_name like concat('%',?,'%')";
        param.add(itemName);
        andFlag = true;
    }
    
    if (maxPrice != null) {
        if (andFlag) {
            sql += " and";
        }
        sql += " price <= ?";
        param.add(maxPrice);
    }
    
    log.info("sql={}", sql);
    return template.query(sql, itemRowMapper(), param.toArray());
}
```
>ItemSearchCond은 Item을 검색할때의 조건을 받는 DTO임

**문제점:**
- 문자열 연결로 SQL 구성 (보기 안 좋음)
- `and` 플래그 관리 필요
- 파라미터 순서를 직접 관리해야 함
- 조건이 많아지면 복잡도 급증

<br>

### 정리
* JdbcTemplate은 동적 쿼리 작성이 매우 번거로움
* JdbcTemplate의 기본 방식(`?` 순서 기반)으로는 파라미터 순서 실수 가능이 높음
