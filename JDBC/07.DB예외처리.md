# 자바 예외 처리 - 활용 및 실무 원칙

## 개요
* 자바 애플리케이션의 안정성은 예외를 어떻게 다루느냐에 달려 있음.
* 이 문서에서는 자바 예외의 기본 계층 구조를 복습하고, 실무에서 **체크 예외**와 **언체크 예외**를 상황에 맞게 선택하여 유지보수하기 좋은 코드를 작성하는 방법을 다룸. 

<br>

## 자바 예외 계층의 이해
모든 예외는 객체이며, `Throwable`을 최상위 부모로 가짐. 

* 각각 Error와 Exception을 자식으로 가짐.
- **Error**: 메모리 부족(OOM) 등 시스템 수준의 복구 불가능한 상태. 개발자가 직접 잡아서는 안됨. 
- **Exception (체크 예외)**: 컴파일러가 체크하는 예외로, 반드시 잡거나 던져야함. 
- **RuntimeException (언체크 예외)**: 컴파일 시점에 체크하지 않으며, 실행 중 발생함(Exception자식임). 

> **주의**: `Throwable`을 직접 catch 하면 안됨. 복구 불가능한 `Error`까지 함께 잡힐 위험이 있기때문. 

<br>

## 예외 처리의 기본 규칙: '잡거나 던지거나'
예외는 내가 처리할 수 없다면 반드시 호출한 곳으로 던져야함. 

1. **예외 처리**: `try-catch`로 예외를 잡으면 이후 로직은 정상 흐름으로 복구됨. 
2. **예외 던짐**: 처리할 수 없는 예외는 `throws`를 통해 나를 호출한 곳으로 던지게됨. 

<br>

## 체크 예외 vs 언체크 예외 전략

### 컴파일러가 강제하는 안전장치: 체크 예외
`Exception`과 그 하위 예외(RuntimeException 제외)에 해당. 

- **장점**: 개발자가 실수로 예외 처리를 누락하지 않도록 컴파일러가 도와줌. 
- **단점**: 모든 예외를 명시적으로 처리해야 하므로 코드가 번잡해지고, 상위 계층까지 불필요한 의존관계가 발생함. 

### 유연한 예외 처리: 언체크(런타임) 예외
`RuntimeException`을 상속받은 예외. 

- **장점**: 신경 쓰고 싶지 않은 예외를 무시할 수 있으며, 의존관계 결합도를 낮출 수 있음.
- **단점**: 개발자가 실수로 예외 처리를 누락할 수 있으므로 명확한 **문서화**가 필수적임. 

<br>

## [실무 원칙] 언제 무엇을 사용해야 하는가?

### 기본 원칙: "언체크 예외를 기본으로 사용하되 정말 특정케이스에 대하여 체크예외를 사용" 
현대적인 자바 라이브러리와 프레임워크(Spring, JPA 등)는 대부분 언체크 예외를 지향하며 이루어짐. 

| 구분 | 권장 사용 상황 | 비고 |
| :--- | :--- | :--- |
| **체크 예외** | 비즈니스 로직상 반드시 처리해야 하는 중요한 예외 (예: 계좌 이체 실패 <== 이거조차 그저 예시) | 최소화 권장  |
| **언체크 예외** | 복구가 불가능한 시스템 예외, 공통 처리가 필요한 예외 (예: DB 접속 오류) | 기본 전략  |

### 체크 예외의 치명적 한계: 의존성 오염 
리포지토리에서 발생한 `SQLException`(체크 예외)을 서비스와 컨트롤러에 그대로 던지게 되면, 향후 데이터 저장 기술을 JPA혹은 다른 데이터접근 기술로 바꿀 때 모든 계층의 코드를 수정해야 하는 일이 벌어짐. 


<br>

## 예외 전환 시 주의사항: 스택 트레이스 보존
체크 예외를 언체크 예외로 전환할 때는 **반드시 기존 예외(cause)를 포함**해야 함.
<br>(런타임 예외를 상속 하여 직접만든 혹은 만들어진 예외에 체크예외를 파라미터로 넘겨 전환 즉 체크를 언체크로 감싼다고 생각하면 됨.)

```java
public void call() {
    try {
        runSQL();
    } catch (SQLException e) {
        throw new RuntimeSQLException(e); //기존 예외 e를 생성자에 넘겨줌
    }
}
```

* 예외를 전환할때는 기존 예외를 넘겨줘야함 그렇지 않으면 어디서 예외가 터졌는지 확인할 수 있는 방법이 없음.

* log.info("message={}", "message", ex) 이렇게 하면 메시지와 ex에대한 스택트레이스를 로그에 남길 수 있음.
