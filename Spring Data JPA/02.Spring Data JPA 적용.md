# 스프링 데이터 JPA 적용

<br>

## 1단계: SpringDataJpaItemRepository 정의

```java
package hello.itemservice.repository.jpa;

import hello.itemservice.domain.Item;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;
import java.util.List;

public interface SpringDataJpaItemRepository extends JpaRepository<Item, Long> {
    // 메서드 이름으로 쿼리 자동 생성
    List<Item> findByItemNameLike(String itemName);

    List<Item> findByPriceLessThanEqual(Integer price);

    List<Item> findByItemNameLikeAndPriceLessThanEqual(String itemName, Integer price);
    
    // 복잡한 쿼리는 @Query로 직접 작성
    @Query("select i from Item i where i.itemName like :itemName and i.price <= :price")
    List<Item> findItems(@Param("itemName") String itemName, @Param("price") Integer price);
}
```

**선택 기준:**
- **간단한 조건 (1~2개):** 쿼리 메서드 사용
- **복잡한 조건 또는 조인:** @Query로 JPQL 직접 작성

> 조건이 3개 이상이면 메서드명이 너무 길어지므로 @Query 권장

<br>

## 2단계: ItemRepository 어댑터 작성

**왜 있어야하는가?**
* ItemService는 ItemRepository 인터페이스에 의존
* SpringDataJpaItemRepository를 직접 노출하면 기술 의존성이 높아짐
* JpaItemRepositoryV2가 중간에서 어댑터 역할

**구조:**
```
ItemService → ItemRepository (인터페이스)
                  ↓
         JpaItemRepositoryV2 (어댑터)
                  ↓
         SpringDataJpaItemRepository (스프링 데이터 JPA)
                  ↓
                JPA (프록시)
```

**코드:**
```java
package hello.itemservice.repository.jpa;

import hello.itemservice.domain.Item;
import hello.itemservice.repository.ItemRepository;
import hello.itemservice.repository.ItemSearchCond;
import hello.itemservice.repository.ItemUpdateDto;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Repository;
import org.springframework.transaction.annotation.Transactional;
import org.springframework.util.StringUtils;
import java.util.List;
import java.util.Optional;

@Repository
@Transactional
@RequiredArgsConstructor
public class JpaItemRepositoryV2 implements ItemRepository {

    private final SpringDataJpaItemRepository repository;
    
    @Override
    public Item save(Item item) {
        return repository.save(item);
    }
    
    @Override
    public void update(Long itemId, ItemUpdateDto updateParam) {
        Item findItem = repository.findById(itemId).orElseThrow();
        findItem.setItemName(updateParam.getItemName());
        findItem.setPrice(updateParam.getPrice());
        findItem.setQuantity(updateParam.getQuantity());
    }
    
    @Override
    public Optional<Item> findById(Long id) {
        return repository.findById(id);
    }
    
    @Override
    public List<Item> findAll(ItemSearchCond cond) {
        String itemName = cond.getItemName();
        Integer maxPrice = cond.getMaxPrice();
        
        if (StringUtils.hasText(itemName) && maxPrice != null) {
            return repository.findItems("%" + itemName + "%", maxPrice);
        } else if (StringUtils.hasText(itemName)) {
            return repository.findByItemNameLike("%" + itemName + "%");
        } else if (maxPrice != null) {
            return repository.findByPriceLessThanEqual(maxPrice);
        } else {
            return repository.findAll();
        }
    }
}
```

**주요 메서드별 동작:**

| 메서드 | 역할 | 생성되는 SQL |
| --- | --- | --- |
| `save(item)` | 저장 및 수정 | INSERT / UPDATE |
| `findById(id)` | PK로 단건 조회 | SELECT ... WHERE id = ? |
| `update()` | 변경 감지로 수정 | UPDATE (트랜잭션 커밋 시) |
| `findAll()` | 조건별 조회 분기 | SELECT (조건 따라 다름) |

## 3단계: 설정 클래스 작성

```java
package hello.itemservice.config;

import hello.itemservice.repository.ItemRepository;
import hello.itemservice.repository.jpa.JpaItemRepositoryV2;
import hello.itemservice.repository.jpa.SpringDataJpaItemRepository;
import hello.itemservice.service.ItemService;
import hello.itemservice.service.ItemServiceV1;
import lombok.RequiredArgsConstructor;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
@RequiredArgsConstructor
public class SpringDataJpaConfig {

    private final SpringDataJpaItemRepository springDataJpaItemRepository;
    
    @Bean
    public ItemService itemService() {
        return new ItemServiceV1(itemRepository());
    }
    
    @Bean
    public ItemRepository itemRepository() {
        return new JpaItemRepositoryV2(springDataJpaItemRepository);
    }
}
```

**동작 흐름:**
1. SpringDataJpaItemRepository: 스프링이 프록시로 자동 생성
2. JpaItemRepositoryV2: 수동 생성 (어댑터 역할)
3. ItemService: 설정에서 주입받음

<br>

## 4단계: 애플리케이션 실행

# JPA 예외 변환

* JPA는 @Repository에서 스프링예외로 바뀌지만   Spring Data JPA는 <br>
SpringDataJpaItemRepository 여기서 자동으로 변환 시켜주므로 @Repository가 없어도 되지만 붙어있는 경우에 <br>
이미 스프링 추상 예외로 바뀌어서 오기 때문에 그냥 지나감

## 동적 쿼리 문제

**현재 코드 (JpaItemRepositoryV2):**
```java
if (StringUtils.hasText(itemName) && maxPrice != null) {
    return repository.findItems("%" + itemName + "%", maxPrice);
} else if (StringUtils.hasText(itemName)) {
    return repository.findByItemNameLike("%" + itemName + "%");
} else if (maxPrice != null) {
    return repository.findByPriceLessThanEqual(maxPrice);
} else {
    return repository.findAll();
}
```

**문제점:**
1. 조건 4가지 = 메서드 4개 필요
2. 조건이 10개면 메서드도 10개 이상 필요
3. 비효율적이고 유지보수 어려움

**해결책:**
```
스프링 데이터 JPA + Querydsl
```
* Querydsl: 자바 코드로 동적 쿼리를 깔끔하게 작성
* 실무에서는 JPA와 Querydsl이 거의 필수 세트
