# 스프링 데이터 JPA란?

* JPA를 편리하게 사용하도록 도와주는 스프링의 라이브러리
* 순수 JPA의 번거로운 부분(EntityManager를 직접 사용 등)을 간결하게 해결
* 실무에서 JPA + Querydsl이 항상 함께 사용됨

```
JPA: EntityManager 직접 사용, JPQL 직접 작성
    ↓
Spring Data JPA: 인터페이스만으로 자동 구현, 메서드 이름으로 쿼리 생성
```

<br>

# 스프링 데이터 JPA 주요 기능

<br>

## 공통 인터페이스 기능

**JpaRepository 상속 구조:**
```
Repository (스프링 데이터)
    ↑
CrudRepository (기본 CRUD - 스프링, 마이바티스, 몽고 등)
    ↑
PagingAndSortingRepository (페이징, 정렬)
    ↑
JpaRepository (JPA 특화 기능)
```

<br>

```java
public interface ItemRepository extends JpaRepository<Item, Long> {
}
```

**스프링이 자동으로 해주는 일:**
1. 구현 클래스 생성
2. 동적 프록시 기술로 빈 등록 및 구현 클래스 생성
3. 기본 CRUD 메서드 제공

>ItemRepository이라는 인터페이스는 JpaRepository인터페이스를 상속 받음(extends)

<br>

## 쿼리 메서드 기능

* 메서드 이름으로 → 자동으로 JPQL 생성 → SQL 실행
> 메서드 이름을 지을때 규칙이 있으며 그거에 맞추어야함

**문제와 해결:**

| 방식 | 코드 | 장점 | 단점 |
| --- | --- | --- | --- |
| **순수 JPA** | `em.createQuery("select m from Member m where ...").setParameter(...)` | 정확함 | 매번 JPQL 작성 |
| **스프링 데이터 JPA (쿼리 메서드)** | `List<Member> findByUsernameAndAgeGreaterThan(String username, int age)` | 간결함 | 조건이 많으면 메서드명 너무 김 |

<br>

### 쿼리 메서드 규칙

**기본 패턴:**
```
find[조건명]By[엔티티필드]And[엔티티필드]...
```

**메서드 이름 분석 예시:**
```java
// 메서드명
List<Item> findByItemNameLike(String itemName);
                     ↓
// 생성되는 JPQL
select i from Item i where i.itemName like ?


// 메서드명
List<Item> findByPriceLessThanEqual(Integer price);
                     ↓
// 생성되는 JPQL
select i from Item i where i.price <= ?


// 메서드명
List<Item> findByItemNameLikeAndPriceLessThanEqual(String itemName, Integer price);
                     ↓
// 생성되는 JPQL
select i from Item i where i.itemName like ? and i.price <= ?
```

**지원하는 키워드:**

| 기능 | 패턴 | 반환타입 |
| --- | --- | --- |
| 조회 | find…By, read…By, query…By, get…By | List<T>, Optional<T> |
| COUNT | count…By | long |
| EXISTS | exists…By | boolean |
| 삭제 | delete…By, remove…By | long |
| 상한선 | findFirst3, findTop, findTop3 | List<T> |

> 더 자세한 규칙은 Spring Data JPA 공식 문서 참고
