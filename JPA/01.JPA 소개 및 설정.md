# JPA란?

* spring과 JPA는 자바 시장의 주력 기술임
* ORM(Object-Relational Mapping) 기술
* SQL을 직접 작성하지 않고 객체 중심으로 개발(자바의 collection을 컨셉으로 만들었음)

<br>

## JdbcTemplate vs MyBatis vs JPA 비교

| 기술 | SQL 작성 | 동적쿼리 |
| --- | --- | --- |
| **JdbcTemplate** | 개발자 직접 | 문자 연결 방식 |
| **MyBatis** | 개발자 직접 (XML) | `<if>`, `<where>` 태그 |
| **JPA** | JPA가 자동 생성 | Querydsl 사용 |

**JPA의 최대 장점:**
* SQL 작성 불필요 → 개발 속도 대폭 향상
* 객체 지향적 개발 가능
* 동적 쿼리는 Querydsl과 함께 사용(실무에서는 JPA와 Querydsl이 세트로 사용됨)


<br>

# JPA 설정

<br>

## 라이브러리 추가

### build.gradle 의존성

```groovy
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    
    // JPA, 스프링 데이터 JPA 추가
    implementation 'org.springframework.boot:spring-boot-starter-data-jpa'
    
    // H2 데이터베이스
    runtimeOnly 'com.h2database:h2'
    
    compileOnly 'org.projectlombok:lombok'
    annotationProcessor 'org.projectlombok:lombok'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}
```

> `spring-boot-starter-data-jpa`는 다음을 자동으로 포함함:
> - `hibernate-core` : JPA 구현체
> - `jakarta.persistence-api` : JPA 표준 인터페이스
> - `spring-data-jpa` : 스프링 연동 라이브러리 <br>
> (JPA는 인터페이스이며 구현체로 하이버네이트를 사용하고 <br>
 JPA라이브러리 안에 JDBC와 JdbcTemplate가 포함되어있어 네이티브로 작성해야할때는 보통 JDBC를 사용함.)

<br>

## 로그 설정

### application.properties

```properties
# JPA SQL 로그
logging.level.org.hibernate.SQL=DEBUG
logging.level.org.hibernate.type.descriptor.sql.BasicBinder=TRACE
```

**로그 항목:**
* `org.hibernate.SQL=DEBUG` : 하이버네이트가 실행하는 SQL 확인
* `org.hibernate.type.descriptor.sql.BasicBinder=TRACE` : SQL 파라미터 확인

>스프링 부트 3.0 이상은 `org.hibernate.orm.jdbc.bind=TRACE` 사용함
