# JPA 적용 - 엔티티 매핑

<br>

## Item 엔티티 설계

```java
package hello.itemservice.domain;

import lombok.Data;
import javax.persistence.*;

@Data
@Entity
public class Item {
    @Id 
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "item_name", length = 10)
    private String itemName;
    
    private Integer price;
    private Integer quantity;
    
    // JPA는 public 또는 protected 기본 생성자 필수임
    public Item() {}
    
    public Item(String itemName, Integer price, Integer quantity) {
        this.itemName = itemName;
        this.price = price;
        this.quantity = quantity;
    }
}
```

<br>

## 애노테이션 상세 설명

|||
| --- | --- |
| `@Entity` | JPA가 관리하는 객체 (필수) |
| `@Id` | 테이블의 PK와 매핑 |
| `@GeneratedValue(strategy = GenerationType.IDENTITY)` | DB가 PK 생성 (MySQL기준: AUTO_INCREMENT) |
| `@Column(name = "item_name")` | 객체 필드와 DB 컬럼명 매핑 |
| `@Column(length = 10)` | DDL 생성 시 컬럼 길이 (varchar 10) |

<br>

### @Column 생략

**문제:**
* Java: `itemName` (camelCase)
* DB: `item_name` (snake_case)

**해결:**
스프링 부트는 자동으로 `itemName` ↔ `item_name` 변환하므로 `@Column` 생략 가능
>마이바티스와 JdbcTemplate과 같이 별칭 역할이 필요한 경우(이름이 다른경우) @Column사용

```java
@Data
@Entity
public class Item {
    private Long id;
    private String itemName;  // 자동으로 item_name과 매핑됨
    private Integer price;
    private Integer quantity;
}
```

# JPA 동작 상세

<br>

## save() - 저장

```java
public Item save(Item item) {
    em.persist(item);
    return item;
}
```

**JPA가 생성하는 SQL:**
```sql
insert into item (item_name, price, quantity) values (?, ?, ?)
```

**동작:**
1. `em.persist(item)` 호출
2. 트랜잭션 커밋 시점에 INSERT 실행
3. DB가 생성한 PK를 `item.id`에 자동 설정 (IDENTITY 전략)

> ID는 INSERT 이후에 자동으로 설정되므로 쿼리에서 null 또는 default로 처리됨

<br>

## update() - 수정

```java
public void update(Long itemId, ItemUpdateDto updateParam) {
    Item findItem = em.find(Item.class, itemId);
    findItem.setItemName(updateParam.getItemName());
    findItem.setPrice(updateParam.getPrice());
    findItem.setQuantity(updateParam.getQuantity());
}
```

**JPA가 생성하는 SQL:**
```sql
update item set item_name=?, price=?, quantity=? where id=?
```

**JPA 마법:**
* `em.update()` 같은 메서드를 호출하지 않음
* 단순히 객체의 필드를 수정하는 것만으로 UPDATE 실행

**이유:**
* JPA는 트랜잭션 커밋 시점에 변경된 엔티티를 자동으로 감지<br>
  (처음 실행할때의 상태를 스냅샷으로 찍어두었다가 커밋시점의 결과와 비교하여 감지함)
* 변경 감지 기능 활용

<br>

## findById() - 단건 조회

```java
public Optional<Item> findById(Long id) {
    Item item = em.find(Item.class, id);
    return Optional.ofNullable(item);
}
```

**JPA가 생성하는 SQL:**
```sql
select item0_.id as id1_0_0_,
       item0_.item_name as item_nam2_0_0_,
       item0_.price as price3_0_0_,
       item0_.quantity as quantity4_0_0_
from item item0_
where item0_.id=?
```

**동작:**
* PK를 기준으로 조회
* 결과를 자동으로 Item 객체로 변환

> 별칭이 복잡해 보이지만 조인이나 복잡한 쿼리를 처리하기 위한것임

<br>

## findAll() - 목록 조회 (JPQL 사용)

```java
public List<Item> findAll(ItemSearchCond cond) {
    String jpql = "select i from Item i";
    
    // ... 동적 쿼리 구성 ...
    
    TypedQuery<Item> query = em.createQuery(jpql, Item.class);
    // ... 파라미터 바인딩 ...
    return query.getResultList();
}
```

**JPQL 예시:**
```sql
select i from Item i
where i.itemName like concat('%',:itemName,'%')
  and i.price <= :maxPrice
```

<br>

**변환된 SQL:**
```sql
select item0_.id, item0_.item_name, item0_.price, item0_.quantity
from item item0_
where (item0_.item_name like ('%'||?||'%'))
  and item0_.price<=?
```

# JPQL (Java Persistence Query Language)

<br>

## JPQL 개요

**SQL vs JPQL:**
* **SQL:** 테이블을 대상으로 쿼리
* **JPQL:** 엔티티 객체를 대상으로 쿼리

```java
// JPQL - 엔티티 객체 사용
String jpql = "select i from Item i where i.price > 10000";

// 하이버네이트가 자동으로 SQL로 변환
// SQL: select * from item where price > 10000
```

<br>

## 파라미터 바인딩

```java
// JPQL에서 파라미터 선언
String jpql = "select i " +
              "from Item i " +
              "where i.price <= :maxPrice";

// 파라미터 바인딩
TypedQuery<Item> query = em.createQuery(jpql, Item.class);
query.setParameter("maxPrice", 50000);

// 실행
List<Item> result = query.getResultList();
```

<br>

## 동적 쿼리의 문제점

**현재 코드 (JPA):**
```java
String jpql = "select i from Item i";

if (StringUtils.hasText(itemName) || maxPrice != null) {
    jpql += " where";
}

boolean andFlag = false;
if (StringUtils.hasText(itemName)) {
    jpql += " i.itemName like concat('%',:itemName,'%')";
    andFlag = true;
}

if (maxPrice != null) {
    if (andFlag) {
        jpql += " and";
    }
    jpql += " i.price <= :maxPrice";
}
```

**문제:**
* JPA도 동일하게 동적 쿼리가 복잡함
* SQL 문자 연결로 가독성 떨어짐

**해결:**
* Querydsl을 함께 사용하면 깔끔하게 해결 가능
* 실무에서 JPA 사용 시 Querydsl은 거의 세트로 사용됨

# JPA 설정 및 실행

<br>

## JpaConfig 설정

```java
package hello.itemservice.config;

import hello.itemservice.repository.ItemRepository;
import hello.itemservice.repository.jpa.JpaItemRepositoryV1;
import hello.itemservice.service.ItemService;
import hello.itemservice.service.ItemServiceV1;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import javax.persistence.EntityManager;

@Configuration
public class JpaConfig {
    private final EntityManager em;
    
    public JpaConfig(EntityManager em) {
        this.em = em;
    }
    
    @Bean
    public ItemService itemService() {
        return new ItemServiceV1(itemRepository());
    }
    
    @Bean
    public ItemRepository itemRepository() {
        return new JpaItemRepositoryV1(em);
    }
}
```

* EntityManager는 스프링 부트가 자동으로 생성
* 생성자 주입으로 주입받기만 하면 됨

<br>

## 애플리케이션 설정

```java
@Import(JpaConfig.class)
@SpringBootApplication(scanBasePackages = "hello.itemservice.web")
public class ItemServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(ItemServiceApplication.class, args);
    }
}
```
