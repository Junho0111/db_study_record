# MyBatis - 동작과정 및 기능

### 개요

* **MyBatis** 인터페이스로 동작과정
* **MyBatis** 기능

<br>


### 구현체없이 인터페이스로 어떻게 동작하나?

```java
@Mapper
public interface ItemMapper {
    void save(Item item);

    Optional<Item> findById(Long id);

    List<Item> findAll(ItemSearchCond itemSearch);
}
```
**의문점:** ItemMapper 구현체가 없는데 어떻게 동작?

<br>

### 동적 프록시 기술

#### MyBatis 스프링 연동 모듈의 동작 원리

```
애플리케이션 로딩
      ↓
MyBatis가 @Mapper 찾음
      ↓
동적 프록시 기술으로 구현체 자동 생성(이떄 생성된 구현체는 xml에 정의한 쿼리문으로 구현됨)
      ↓
생성된 구현체를 스프링 빈으로 등록
      ↓
의존성 주입 가능
```

<br>

**동작 흐름**
```
MyBatis 스프링 연동 모듈
      ↓
@Mapper 조사
      ↓
ItemMapper 인터페이스 발견
      ↓
동적 프록시 기술 사용
      ↓
ItemMapper 구현체 자동 생성
      ↓
스프링 빈 등록
      ↓
<<proxy>> ItemMapper 구현체
      ↓
의존성 주입
      ↓
MyBatisItemRepository에서 사용 가능
```

<br>

### 실제 동작 확인

```java
@Slf4j
@Repository
@RequiredArgsConstructor
public class MyBatisItemRepository implements ItemRepository {
    private final ItemMapper itemMapper;
    
    @Override
    public Item save(Item item) {
        log.info("itemMapper class={}", itemMapper.getClass());
        // 출력: itemMapper class=class com.sun.proxy.$Proxy66
        
        itemMapper.save(item);
        return item;
    }
}
```

```
로그 => itemMapper class=class com.sun.proxy.$Proxy66
```

>JDK 동적 프록시가 적용됨 (com.sun.proxy.$Proxy...)

<br>

### 매퍼 구현체

MyBatis가 자동 생성하는 구현체는:

|||
| --- | --- |
| **SQL 실행** | XML의 SQL을 찾아서 실행 |
| **파라미터 바인딩** | `@Param`과 `#{}` 자동 매핑 |
| **결과 매핑** | `resultType` 기반 자동 매핑 |
| **예외 변환** | MyBatis 예외 → Spring 예외 변환 |
| **트랜잭션 연동** | 스프링 트랜잭션과 자동 동기화 |

<br>

### 예외 변환

```
MyBatis 에러 발생
       ↓
MyBatis 구현체가 자동으로 변환
       ↓
Spring DataAccessException으로 변환
       ↓
서비스 계층에서 안전하게 처리
```

이는 JdbcTemplate과 동일한 예외 변환 기능임

## MyBatis 기능 정리

MyBatis의 최고의 기능 => 동적 SQL 작성의 편리함

**제공되는 동적 SQL 태그:**
*`<if>` : 조건부 포함
* `<choose>`, `<when>`, `<otherwise>` : switch 문
* `<trim>`, `<where>`, `<set>` : 문자 제거 등
* `<foreach>` : 반복 처리

<br>

### if 태그

#### 사용 예시

```xml
<select id="findActiveBlogWithTitleLike" resultType="Blog">
    SELECT * FROM BLOG
    WHERE state = 'ACTIVE'
    <if test="title != null">
        AND title like #{title}
    </if>
</select>
```

**동작:**
* `test` 속성의 조건이 `true`면 내부 내용 포함
* 조건이 `false`면 내부 내용 제외

**OGNL 표기법:**
```xml
<if test="title != null">                              <!-- null 체크 -->
<if test="title != null and title != ''">            <!-- null 과 공백 체크 -->
<if test="price > 10000">                              <!-- 숫자 비교 -->
<if test="list != null and list.size() > 0">         <!-- 컬렉션 체크 -->
```

<br>

### choose, when, otherwise 태그

#### 사용 예시

#### Java의 switch 문과 동일한 구조

```xml
<select id="findActiveBlogLike" resultType="Blog">
    SELECT * FROM BLOG WHERE state = 'ACTIVE'
    <choose>
        <when test="title != null">
            AND title like #{title}
        </when>
        <when test="author != null and author.name != null">
            AND author_name like #{author.name}
        </when>
        <otherwise>
            AND featured = 1
        </otherwise>
    </choose>
</select>
```

* 첫 번째 `when`이 true → 그 내용 실행, 나머지 무시
* 첫 번째가 false → 두 번째 `when` 검사
* 모든 `when` false → `otherwise` 실행

**케이스 분석:**

| title | author | 결과 |
| --- | --- | --- |
| null | null | `AND featured = 1` |
| 값 | null | `AND title like ?` |
| null | 값 | `AND author_name like ?` |
| 값 | 값 | `AND title like ?` (첫 번째만) |

<br>

### trim, where, set 태그

#### 문제: 조건에 따른 AND/OR 관리의 어려움

#### 사용 예시

```xml
<!-- 문제 있는 코드 -->
<select id="findActiveBlogLike" resultType="Blog">
    SELECT * FROM BLOG
    WHERE
    <if test="state != null">
        state = #{state}
    </if>
    <if test="title != null">
        AND title like #{title}
    </if>
</select>
```

**문제 케이스:** 모든 조건이 false
```sql
SELECT * FROM BLOG
WHERE
-- WHERE 다음이 아무것도 없음 (오류!)
```

**문제 케이스:** title만 true
```sql
SELECT * FROM BLOG
WHERE
AND title like 'someTitle'
-- WHERE 다음이 AND (오류!)
```

<br>

#### `<where>` 태그 (권장)

```xml
<select id="findActiveBlogLike" resultType="Blog">
    SELECT * FROM BLOG
    <where>
        <if test="state != null">
            state = #{state}
        </if>
        <if test="title != null">
            AND title like #{title}
        </if>
        <if test="author != null and author.name != null">
            AND author_name like #{author.name}
        </if>
    </where>
</select>
```

**`<where>`태그 기능**

| 상황 | 결과 |
| --- | --- |
| 조건 없음 | WHERE 제거 |
| 첫 AND | WHERE로 변환 |
| 조건 있음 | WHERE 추가 |

**모든 조건 false**
```sql
SELECT * FROM BLOG
-- <where>가 제거됨 
```

**title만 true**
```sql
SELECT * FROM BLOG
WHERE AND title like 'someTitle'
-- <where>가 첫 AND를 WHERE로 변환 
→ WHERE title like 'someTitle'
```

**state와 title 둘다 true**
```sql
SELECT * FROM BLOG
WHERE state = 'ACTIVE' AND title like 'someTitle'
```

<br>

#### `<trim>` 태그 

```xml
<!-- <where>과 동일한 기능을 하는 trim -->
<select id="findActiveBlogLike" resultType="Blog">
    SELECT * FROM BLOG
    <trim prefix="WHERE" prefixOverrides="AND |OR ">
        <if test="state != null">
            state = #{state}
        </if>
        <if test="title != null">
            AND title like #{title}
        </if>
    </trim>
</select>
```

**`<trim>`**

|||
| --- | --- |
| `prefix="WHERE"` | 시작에 WHERE 추가 |
| `prefixOverrides="AND \|OR "` | 시작에 AND/OR 있으면 제거 |
| `suffix="..."` | 끝에 추가 (생략 가능) |
| `suffixOverrides="..."` | 끝에서 제거 (생략 가능) |

<br>

#### `<set>` 태그 (UPDATE용)

```xml
<update id="updateAuthor">
    UPDATE author
    <set>
        <if test="username != null">
            username = #{username},
        </if>
        <if test="password != null">
            password = #{password},
        </if>
        <if test="email != null">
            email = #{email},
        </if>
    </set>
    WHERE id = #{id}
</update>
```

**`<set>`의 기능:**
* SET 키워드 자동 추가
* 마지막 쉼표(,) 자동 제거
* UPDATE 문의 WHERE/SET 혼동 방지

**동작:**

```sql
<!-- username만 업데이트 -->
UPDATE author
SET username = #{username}
WHERE id = #{id}
<!-- <set>이 마지막 쉼표 제거함 -->
```

<br>

### foreach 태그

#### 컬렉션을 반복 처리 (IN 쿼리)

```xml
<select id="selectPostIn" resultType="domain.blog.Post">
    SELECT *
    FROM POST P
    <where>
        <foreach item="item" index="index" collection="list"
                 open="ID in (" separator="," close=")" nullable="true">
            #{item}
        </foreach>
    </where>
</select>
```

**사용 예시:**
```java
mapper.selectPostIn(Arrays.asList(1, 2, 3, 4, 5));
```

**생성되는 SQL:**
```sql
SELECT * 
FROM POST P
WHERE ID in (1,2,3,4,5)
```

<br>

#### `<foreach>` 상세

| 속성 | 설명 | 예시 |
| --- | --- | --- |
| `item` | 반복 변수명 | `#{item}` 에서 사용 |
| `index` | 인덱스 (0부터) | `#{index}` |
| `collection` | 반복할 컬렉션 명 | list, items 등 |
| `open` | 시작 문자 | `"("` |
| `separator` | 구분자 | `","` |
| `close` | 끝 문자 | `")"` |
| `nullable` | null 허용 | "true"/"false" |

#### 다양한 예시

**Map 반복:**
```xml
<foreach item="item" index="key" collection="map" open="(" separator="," close=")">
    #{key}=#{item}
</foreach>
<!-- #{key}=값1, #{key}=값2, ... -->
```

**배열 반복:**
```xml
<foreach item="item" collection="array" open="(" separator="," close=")">
    #{item}
</foreach>
```

**여러 열 INSERT:**
```xml
<insert id="insertBatch">
    INSERT INTO table (id, name) VALUES
    <foreach item="item" collection="list" separator=",">
        (#{item.id}, #{item.name})
    </foreach>
</insert>
```

**생성되는 SQL:**
```sql
INSERT INTO table (id, name) VALUES (1, 'a'), (2, 'b'), (3, 'c')
```

## MyBatis - 기타 기능

### 애노테이션으로 SQL 작성

#### 간단한 쿼리는 XML 없이 인터페이스에서 작성

```java
@Mapper
public interface ItemMapper {
    @Select("select id, item_name, price, quantity from item where id=#{id}")
    Optional<Item> findById(Long id);
    
    @Insert("insert into item (item_name, price, quantity) values (#{itemName}, #{price}, #{quantity})")
    void save(Item item);
    
    @Update("update item set item_name=#{itemName}, price=#{price}, quantity=#{quantity} where id=#{id}")
    void update(Item item);
    
    @Delete("delete from item where id=#{id}")
    void delete(Long id);
}
```

**제공되는 애노테이션:**
* `@Select` : SELECT
* `@Insert` : INSERT
* `@Update` : UPDATE
* `@Delete` : DELETE

**주의:**
* 동적 쿼리 작성 불가 (간단한 경우만)
* 여러 줄 SQL은 가독성 떨어짐

**권장:**
```
┌─ 간단한 CRUD → 애노테이션
│
기술 선택
│
└─ 동적 쿼리 필요 → XML
```
>보통 간단한 CRUD의 경우 JdbcTemplate를 쓰며, 복잡한 동적 쿼리가 들어가는 경우에 MyBatis를 사용함
