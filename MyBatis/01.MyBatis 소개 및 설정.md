# MyBatis 소개 및 설정

### 개요

* **MyBatis란?** SQL Mapper 
* 기존 JdbcTemplate의 동적 쿼리 문제를 해결 함

<br>

## JdbcTemplate vs MyBatis

MyBatis는 쿼리문이 복잡하고 동적인 부분에 대해서 강점을 보이며,<br>
xml에 sql문을 작성하듯이 자연스럽게 사용하면 되어 보는 사람으로 하여금 가독성이 높음

#### SQL 여러 줄 작성 비교

**JdbcTemplate 방식 - 문자 연결 필요**
```java
String sql = "update item " +
    "set item_name=:itemName, price=:price, quantity=:quantity " +
    "where id=:id";
```

**MyBatis 방식 - XML에서 SQL쿼리문을 짜듯이 자연스럽게 작성**
```xml
<update id="update">
    update item
    set item_name=#{itemName},
        price=#{price},
        quantity=#{quantity}
    where id = #{id}
</update>
```

<br>

#### 동적 쿼리 비교

**JdbcTemplate**
```java
.
.
.

String sql = "select id, item_name, price, quantity from item";

if (StringUtils.hasText(itemName) || maxPrice != null) {
    sql += " where";
}

boolean andFlag = false;

if (StringUtils.hasText(itemName)) {
    sql += " item_name like concat('%',:itemName,'%')";
    andFlag = true;
}

if (maxPrice != null) {
    if (andFlag) {
        sql += " and";
    }
    sql += " price <= :maxPrice";
}

return template.query(sql, param, itemRowMapper());
```

**MyBatis**
```xml
<select id="findAll" resultType="Item">
    select id, item_name, price, quantity
    from item
    <where>
        <if test="itemName != null and itemName != ''">
            and item_name like concat('%',#{itemName},'%')
        </if>
        <if test="maxPrice != null">
            and price &lt;= #{maxPrice}
        </if>
    </where>
</select>
```
> where태그에서 조건 충족시 where를 붙이며 안의 and 또한 알아서 관리하여 sql문법 오류가 나지 않음 <br>
ex) test="itemName != null and itemName != '' 이 조건 만족시 where뒤에 and를 때고 <br>
item_name like concat('%',#{itemName},'%') 이 부분만 추가되며 만약 두 조건 모두 충족 시<br>
뒤에 오는 쿼리문의 and는 그대로 둠

<br>

## MyBatis 설정

### 라이브러리 추가

#### build.gradle 의존성 추가

```groovy
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-thymeleaf'
    implementation 'org.springframework.boot:spring-boot-starter-web'
    
    // JdbcTemplate 추가
    implementation 'org.springframework.boot:spring-boot-starter-jdbc'
    
    // MyBatis 추가
    implementation 'org.mybatis.spring.boot:mybatis-spring-boot-starter:2.2.0'
    
    // H2 데이터베이스 추가
    runtimeOnly 'com.h2database:h2'
    
    compileOnly 'org.projectlombok:lombok'
    annotationProcessor 'org.projectlombok:lombok'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
    testCompileOnly 'org.projectlombok:lombok'
    testAnnotationProcessor 'org.projectlombok:lombok'
}
```
>잘보면 spring이 제공하는 jdbc의 경우 org뒤에 바로 스프링이 붙어 스프링이 알아서 버전 관리 등 <br>
알아서 관리를 해주지만 MyBatis의 경우 spring이 제공하는 것이 아니기때문에 mybatis뒤에 spring이 붙어 있는걸 확인 할 수 있고, 버전 또한 명시를 해줘야함 <br>
(패키지명에 ibatis라고 나와있는건 옛날 이름이며 현재는 mybatis)

<br>

#### 버전 선택 기준

| 스프링 부트 버전 | MyBatis 버전 | 설정 |
| --- | --- | --- |
| 2.x (권장) | 2.2.0 | `implementation 'org.mybatis.spring.boot:mybatis-spring-boot-starter:2.2.0'` |
| 3.0 이상 | 3.0.3 | `implementation 'org.mybatis.spring.boot:mybatis-spring-boot-starter:3.0.3'` |
| 4.0 이상 | 4.0.1 | `implementation 'org.mybatis.spring.boot:mybatis-spring-boot-starter:4.0.1'` |

**주의:** 버전 정보를 명시하는 이유는 스프링 부트가 공식적으로 관리하지 않는 라이브러리이기 때문

<br>

### 포함되는 라이브러리

`mybatis-spring-boot-starter` 추가 시 자동으로 다음이 포함됨:

- **mybatis-spring-boot-starter:** MyBatis를 스프링 부트에서 편리하게 사용 가능하게 하는 시작 라이브러리
- **mybatis-spring-boot-autoconfigure:** MyBatis와 스프링 부트 자동 설정 라이브러리
- **mybatis-spring:** MyBatis와 스프링을 연동하는 라이브러리
- **mybatis:** 코어 MyBatis 라이브러리

<br>

### 설정 항목 상세 설명

#### mybatis.type-aliases-package
```properties
mybatis.type-aliases-package=hello.itemservice.domain
```

* MyBatis에서 타입 정보를 사용할 때 패키지 이름을 자동으로 인식
* 지정한 패키지와 그 하위 패키지 모두 자동 인식
* 예: `resultType="Item"` (패키지명 생략 가능)

**여러 패키지 지정:**
```properties
mybatis.type-aliases-package=hello.itemservice.domain,hello.itemservice.dto
```

<br>

#### mybatis.configuration.map-underscore-to-camel-case
```properties
mybatis.configuration.map-underscore-to-camel-case=true
```

* JdbcTemplate의 `BeanPropertyRowMapper`처럼 언더스코어를 카멜케이스로 자동 변환
* DB: `item_name` → Java: `itemName` 자동 매핑
* 관례의 불일치 문제 자동 해결

```sql
select item_name as itemName from item  -- 별칭 필수
```

**활성화 후:**
```sql
select item_name from item  -- 별칭 불필요
```

<br>

#### logging.level.hello.itemservice.repository.mybatis=trace
```properties
logging.level.hello.itemservice.repository.mybatis=trace
```

* MyBatis에서 실행되는 SQL과 파라미터를 콘솔에 출력
* **TRACE** 레벨: 가장 상세한 로그 (파라미터 값까지 표시)

**로그 예시:**
```
==>  Preparing: select id, item_name, price, quantity from item where id = ?
==> Parameters: 1(Long)
<==    Columns: ID, ITEM_NAME, PRICE, QUANTITY
<==        Row: 1, itemA, 10000, 100
```

<br>

### 관례의 불일치 문제

**자바 (camelCase):**
```java
itemName, maxPrice, memberName
```

**DB (snake_case):**
```sql
item_name, max_price, member_name
```

#### 자동 변환의 마법

설정 활성화 시:
```sql
select item_name, price, quantity from item
-- 자동으로 다음과 같이 처리됨:
-- item_name → setItemName()
-- price → setPrice()
-- quantity → setQuantity()
```

#### 완전히 다른 이름의 경우 - 별칭 사용

```sql
-- DB에는 member_name, 객체에는 username
select member_name as username from members

-- MyBatis가 자동으로 username 필드에 매핑
```

### XML 파일 위치 설정

기본적으로 `ItemMapper.java`와 같은 위치에 `ItemMapper.xml` 배치:
```
src/main/java/hello/itemservice/repository/mybatis/ItemMapper.java
src/main/resources/hello/itemservice/repository/mybatis/ItemMapper.xml
```

**다른 위치에 두고 싶은 경우:**
```properties
mybatis.mapper-locations=classpath:mapper/**/*.xml
```

이렇게 설정하면:
* `resources/mapper` 폴더의 모든 XML이 자동 인식
* 파일 이름은 자유롭게 설정 가능
* 테스트 설정도 함께 수정 필요
